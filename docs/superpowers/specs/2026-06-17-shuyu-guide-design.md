# 书余 & 凡例 功能设计

**日期**: 2026-06-17
**状态**: 已确认

---

## 一、概述

在导航栏增加"书余"和"凡例"两个模块，替换原有的"分类浏览"和"最新上架"占位。

- **书余**: 用户在阅读过程中摘录原文、记录心得/疑问，可选择私密沉淀或公开投递
- **凡例**: 静态页面，包含网站使用指南和社区规范

---

## 二、书余

### 2.1 功能定位

一句话定义：用户在阅读过程中，随时摘录原文、记录灵光乍现的疑惑与心得，并自主选择"藏于思余"或"发往余音"。

核心三要素：
1. **原文锚定** — 必须绑定书本与段落（CFI定位）
2. **类型标签** — 疑问 或 心得
3. **隐私开关** — 默认仅自己可见（思余），勾选后公开（余音）

流转逻辑：所有记录先进入「思余」，用户一键即可"送入余音"参与广场讨论。

### 2.2 用户操作路径

| 步骤 | 动作 | 对应页面/组件 | 关键细节 |
|------|------|-------------|---------|
| 1. 触发 | 用户在阅读页选中一段文字 | BookReader.vue | 长按/悬浮弹出操作栏：复制 / 划线 / 记书余 |
| 2. 记录 | 点击"记书余"，弹出撰写浮层 | 半屏浮层（不打断阅读） | 自动填入引用原文，下方留白写心得/疑问 |
| 3. 标记 | 选择类型：疑问 或 心得 | 浮层顶部的Tab切换 | 选"疑问"时，文本框提示改为"你哪里困惑？" |
| 4. 选择归宿 | 默认存入思余，勾选"同步至余音"则公开 | 浮层底部开关 | 默认关闭，鼓励用户先私密沉淀 |
| 5. 完成 | 点击"留墨"按钮 | 成功提示 + 微动效 | 段落下方出现淡灰色角标，表示此处有书余 |

### 2.3 页面结构

**书余主页（/notes）**：双Tab结构

- **思余 Tab** — 个人手记管理
  - 按书籍分组展示
  - 每条手记显示：原文引用 / 内容 / 类型标签 / 时间
  - 操作：送入余音、编辑、删除
  - 筛选：按类型、按书籍

- **余音 Tab** — 公开手记广场
  - 每页随机展示公开手记
  - 每条显示：原文引用 / 内容 / 作者 / 书籍 / 时间
  - 互动：回复（树形嵌套）、点赞、举报
  - 候选池构成（对用户透明）：
    - 40% 从用户书架中书籍的公开手记中随机捞取
    - 60% 从全站其余公开手记中随机捞取
    - 混合后随机打乱

### 2.4 阅读器改动

在 BookReader.vue 中增加：
- 选中文字事件监听（epubjs rendition.on('selected')）
- 操作栏浮泡（复制 / 划线 / 记书余），定位在选区上方
- 撰写半屏浮层（原文引用只读区 + 类型切换 + 文本输入 + 同步至余音开关 + 留墨按钮）
- 已有批注渲染（划线样式 + 段落角标）

## 三、凡例

### 3.1 定位

静态页面（/guide），由开发者维护内容，无需管理后台。

### 3.2 内容大纲

**使用指南**
1. 开始阅读 — 如何浏览书籍、进入阅读器、调整字号
2. 记书余 — 选中文字 > 记书余 > 写心得/疑问 > 留墨
3. 思余与余音 — 私密沉淀与公开投递的区别与操作
4. 写书评 — 在书籍详情页发表/回复书评
5. 书架与收件箱 — 收藏书籍、查看通知

**社区规范**
1. 书评准则 — 尊重原作、言之有物、禁止人身攻击
2. 手记发布要求 — 原创、与原文相关、不偏离阅读讨论
3. 举报规则 — 可举报内容类型、处理流程说明
4. 账号规范 — 禁止行为与账号注销说明

## 四、数据库

### 4.1 新增 notes 表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT (PK, AUTO) | 主键 |
| user_id | BIGINT (FK) | 作者 |
| book_id | BIGINT (FK) | 所属书籍 |
| content | TEXT | 批注内容 |
| selected_text | VARCHAR(500) | 选中的原文片段 |
| cfi | VARCHAR(500) | EPUB CFI 位置标识 |
| type | VARCHAR(20) | QUESTION / INSIGHT |
| is_published | BOOLEAN | 是否已投递，默认 false |
| parent_id | BIGINT | 回复目标手记ID |
| root_id | BIGINT | 根手记ID（回复线程） |
| like_count | INT | 点赞数，默认 0 |
| reply_count | INT | 回复数，默认 0 |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |

### 4.2 新增 note_likes 表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT (PK, AUTO) | 主键 |
| note_id | BIGINT (FK) | 手记ID |
| user_id | BIGINT (FK) | 点赞用户 |
| UNIQUE(note_id, user_id) | — | 防重复点赞 |

### 4.3 扩展现有 reports 表

增加 `note_id` 字段（BIGINT, nullable），与现有 `review_id` 并存。
增加 `target_type` 字段（VARCHAR(20)），取值 `review` / `note`，区分举报对象类型。

## 五、后端 API

### 5.1 NoteController

```
POST   /api/notes                    创建手记
GET    /api/notes/mine               我的思余列表（分页，可按书籍/类型筛选）
GET    /api/notes/book/:bookId/mine  某本书下我的思余
PUT    /api/notes/:id                编辑手记内容
DELETE /api/notes/:id                删除手记
POST   /api/notes/:id/publish        投递至余音
POST   /api/notes/:id/unpublish      撤回至思余
GET    /api/notes/public             余音广场（随机池，分页）
GET    /api/notes/book/:bookId/public 某本书的公开手记
POST   /api/notes/:id/reply          回复手记
POST   /api/notes/:id/like           点赞/取消点赞
```

### 5.2 扩展现有 Controller

```
ReportController:
  POST /api/reports  扩展支持 note_id + target_type: "note"

AdminReportController:
  兼容 note 类型举报的查看与处理
```

### 5.3 余音广场随机池算法

```
GET /api/notes/public?page=N&size=M

1. 获取当前用户书架中的书籍ID列表
2. 从书架书籍的公开手记中随机捞取 total * 40% 条
3. 从非书架书籍的公开手记中随机捞取 total * 60% 条
4. 合并后随机打乱，返回分页结果
5. 若用户未登录、无书架数据、或书架书籍无公开手记，100% 全站随机
```

## 七、错误处理

- 批注保存失败：toast 提示，浮层不关闭，允许重试
- 投递失败：toast 提示，状态不回滚
- 余音加载失败：空状态占位图 + 重试按钮
- CFI 定位失败：批注仍保存，仅不显示段落角标
- EPUB 内容变更导致 CFI 偏移：批注按章节分组展示，CFI 作为尽力定位

## 八、非功能需求

- 手记创建/编辑需登录验证（复用现有 JWT 拦截）
- 回复/点赞需登录验证
- 余音广场支持未登录浏览（只读）
- 思余 Tab 需登录
- 凡例无需登录
