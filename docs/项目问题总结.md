# 书海（云图书馆）项目问题总结

> 记录项目启动以来遇到的所有问题及解决方案，分为「错误修改」和「功能实现」两个方向。

---

## 一、错误修改

### 1. Java 26 兼容性：Lombok 注解失效

**为什么遇到**：项目使用 Java 26，Lombok 的注解处理器在该版本上不兼容，导致编译期无法生成 getter/setter/builder 代码。

**怎么解决的**：全局移除 Lombok 依赖，所有实体类、DTO 类手动编写 getter/setter/constructor/builder 方法。

**涉及提交**：`67ea708`, `48074b0`, `69f7247` 等

---

### 2. Spring DI 失败：FileService 缺少无参构造器

**为什么遇到**：Spring 通过反射创建 Bean 时需要无参构造器，手动改造时遗漏了。

**怎么解决的**：为 `FileService` 添加显式无参构造器，确保 Spring DI 容器正常初始化。

**涉及提交**：`69f7247`, `ed7f12c`

---

### 3. JPQL 语法兼容：CONCAT 函数在不同数据库间不一致

**为什么遇到**：MySQL 和 H2（测试用）对 JPQL 的 `CONCAT` 函数行为不同，测试时报 SQL 语法错误。

**怎么解决的**：使用 JPQL 标准的 `CONCAT` 写法，同时将 `ApiResponse` 改回手动实现以保证跨数据库兼容。

**涉及提交**：`48074b0`

---

### 4. CORS 跨越与 JWT 白名单遗漏

**为什么遇到**：
- 前端（reader-app:5174 / admin-app:5178 / login-app:5176）请求后端 API 时，浏览器拦截跨域请求
- 新加的 API 路径（如 `/api/reports`、`/api/reviews`、`/api/notifications`）未加入 JWT 白名单，导致请求被拦截，用户身份 `reporterId` 为 null

**怎么解决的**：
- 后端 CORS 配置中添加所有前端端口
- 每次新增 API 模块时，将对应路径加入 `JwtFilter` / `SecurityConfig` 白名单

**涉及提交**：`0d5cede`, `41f6b85`, `5db427b`, `ae9f281`, `2a057dc`

---

### 5. Token 过期后前端无响应 / 错误重定向

**为什么遇到**：
- 后端返回 401/403 时，前端的 axios 拦截器未统一处理，导致页面空白或错误堆积
- 退出登录后跳转到 login-app 而非 reader-app 首页，用户体验不连贯

**怎么解决的**：
- 前端 401 拦截器统一触发：清除本地 token → 弹窗提示 → 重定向至登录页
- 退出登录改为 `window.location.href` 硬刷新跳转至 reader-app 首页，确保 header 状态同步

**涉及提交**：`35171d9`, `8526896`, `b3ad4e6`, `1eaa625`

---

### 6. EPUB iframe sandbox 阻止文本选中事件

**为什么遇到**：epub.js 阅读器底层使用 iframe 渲染电子书内容，iframe 的 `sandbox` 属性默认阻止了文本选择事件，导致批注功能无法获取选中文本。

**怎么解决的**：
- 初始尝试：用 `MutationObserver` 监听 DOM 变化并移除 sandbox 属性，但 DOM 结构变化时机不可控
- 最终方案：利用 `rendition.hooks.render` 钩子，在每次渲染完成后直接操作 iframe 的 `contentDocument`，移除 sandbox 属性

**涉及提交**：`f4b0b69`, `a828645`

---

### 7. 批注高亮不可见

**为什么遇到**：epub.js 渲染的高亮层渲染时机晚于内容渲染，且高亮的 CSS 层级和颜色设置不当。

**怎么解决的**：通过 `rendition.hooks.content` 注入高亮 CSS 样式，提高对比度；同时为批注面板添加取消按钮。

**涉及提交**：`f4f51bf`

---

### 8. 导航栏切换页面时闪烁（BannerHeader 重复渲染）

**为什么遇到**：`BannerHeader` 组件被放在各个页面组件内部，每次路由切换时组件销毁重建，导致视觉闪烁。

**怎么解决的**：将 `BannerHeader` 提升至 `App.vue` 根层级，跨路由保持常驻，仅在需要隐藏的页面（如阅读器全屏模式）通过路由判断控制显隐。

**涉及提交**：`e695da3`

---

### 9. 举报重复提交报 500 错误

**为什么遇到**：同一用户对同一笔记重复举报时，数据库唯一约束触发 `DataIntegrityViolationException`，但后端未捕获该异常，直接返回 500。

**怎么解决的**：
- 前端校验改为后端唯一约束兜底
- 后端 Service 层捕获 `DataIntegrityViolationException`，转换为友好的"您已举报过此内容"提示
- 在错误信息中包含 `noteId` 和 `reporterId` 辅助调试

**涉及提交**：`728da29`, `b5ee34b`, `6065278`

---

### 10. 笔记通知中 noteId 缺失

**为什么遇到**：创建通知时使用了旧的 `createNotification` 方法，未包含 `noteId` 字段，导致"通知 → 笔记详情"的跳转链路断裂。

**怎么解决的**：添加专用的 `createNoteNotification` 方法，确保笔记相关通知始终携带 `noteId`。

**涉及提交**：`94e880a`, `3732baf`

---

### 11. 余音（Yuyin）星空图颜色过深

**为什么遇到**：余音可视化使用了深色主题配色，与"书余"（Siyu）的浅色主题不一致，视觉割裂。

**怎么解决的**：将余音的节点颜色、连线颜色、背景色调整为与书余一致的浅暖色系，移除深色模式对比度。

**涉及提交**：`7d8d367`

---

### 12. 余音节点点击外部关闭、切换节点不刷新

**为什么遇到**：点击余音节点空白区域时弹窗不关闭，点击另一个节点时详情面板不更新。

**怎么解决的**：添加全局点击事件监听判断点击目标是否在面板外；节点点击时检查是否同一节点，不同则刷新详情数据。

**涉及提交**：`6fa0cb7`

---

### 13. 登录页雨滴 Canvas HiDPI 坐标空间错误

**为什么遇到**：添加 `devicePixelRatio` 缩放时，Canvas 画布尺寸设为物理像素（`width × dpr`），同时调用 `ctx.scale(dpr, dpr)` 缩放坐标系，但雨滴坐标使用的是物理像素值而非 CSS 像素值，导致 Retina 屏幕上大量雨滴绘制在可视区域之外。

**怎么解决的**：统一坐标系——
- `onMounted` 中用 `canvas.offsetWidth/offsetHeight`（CSS 像素）初始化雨滴
- `handleResize` 中用 `contentRect` 的 CSS 像素值初始化雨滴
- `update` 中用 `canvas.offsetWidth` 作为随机 x 坐标的上限

**涉及提交**：`946dc6c`

---

### 14. 颜色解析在动画热循环中重复计算

**为什么遇到**：雨滴 Canvas 每帧（约 16ms）对 45+ 条雨线各执行一次 `props.color.split(',').map(parseInt)`，每秒约 2700 次字符串解析。

**怎么解决的**：将颜色解析提升到 `draw()` 函数的 `for` 循环之外，每帧只解析一次，存入 `[r, g, b]` 变量后在循环内复用。

**涉及提交**：`8d36ddb`

---

### 15. 雨滴重置时使用了过时的透明度/粗细值

**为什么遇到**：参数调整时，`createDrop` 中的 opacity/thickness 范围更新了，但 `update` 函数中的重置逻辑未同步更新，导致重置后的雨滴视觉参数不一致。

**怎么解决的**：统一 `createDrop` 和 `update` 中所有属性随机值范围，后续通过 `replace_all` 确保两处同步更新。

**涉及提交**：修复合并于多次 tweak 提交中

---

### 16. 图书封面 404

**为什么遇到**：上传文件存储路径使用了相对于 JVM 工作目录的路径，在不同运行环境下解析不一致。

**怎么解决的**：改用 `user.home` 作为上传根目录的基准路径，统一各环境的文件访问。

**涉及提交**：`bfe7e24`

---

### 17. 空分类 ID 导致 500 错误

**为什么遇到**：新增图书时，前端表单的分类下拉框默认空值，后端 `@RequestPart` 解析空字符串时类型转换失败。

**怎么解决的**：将非文件的表单字段改用 `@RequestParam` 接收，前端添加默认值和空值校验。

**涉及提交**：`3587977`, `f51efc0`

---

### 18. 测试失败：缺少 FileUploadConfig 导入和 Bearer Token

**为什么遇到**：添加 JWT 鉴权后，原有测试未更新——测试类未导入 `FileUploadConfig`，且请求未携带 Bearer token。

**怎么解决的**：在测试类的 `@Import` 中添加 `FileUploadConfig`，在请求头中添加 `Authorization: Bearer <token>`。

**涉及提交**：`777a2ec`, `96b74ba`

---

### 19. EPUB 上传封面页干扰阅读流程

**为什么遇到**：EPUB 文件的第一页通常是封面图片页，epub.js 将其作为正文第一章渲染，影响阅读体验。

**怎么解决的**：在解析 EPUB 章节时跳过封面页，从第一个正文章节开始。

**涉及提交**：`8351e17`

---

### 20. 空指针异常：`@RequestPart` 与文件上传混用

**为什么遇到**：`@RequestPart` 用于接收非文件的简单字段时，某些 Spring 版本或配置下会触发空指针。

**怎么解决的**：非文件字段改用 `@RequestParam`，仅在文件字段使用 `@RequestPart`。

**涉及提交**：`f51efc0`, `922f0c1`

---

## 二、功能实现

### 1. JWT 用户认证系统

**技术栈**：Spring Security、JJWT（Java JWT 库）、Vue 3 Composition API、axios 拦截器、localStorage

**实现方式**：
- 后端：`JwtUtil` 负责 Token 生成与验证；`JwtFilter` 作为 OncePerRequestFilter 拦截所有 `/api/**` 请求，校验 Authorization Header；`SecurityConfig` 配置白名单路径和 CORS
- 前端：`login-app`（Vue 3 + Vite）独立部署，提供登录/注册表单；reader-app 和 admin-app 通过 axios 拦截器在请求头注入 Token，401 时自动清除 Token 并跳转登录页
- 用户实体包含 `role` 字段区分 `USER` / `ADMIN`，登录后根据角色跳转至不同前端应用

---

### 2. 图书目录系统

**技术栈**：Spring Boot JPA、MySQL、Vue 3、Vite

**实现方式**：
- 实体层：`Book`（含标题、作者、ISBN、简介、分类等）、`Category`（树形结构，parent_id 自引用）
- `CategoryService.buildTree()` 递归构建分类树，支持多级子分类
- REST API：分页查询、关键词搜索、分类筛选、单本详情
- 前端 reader-app：`BookCard` 网格展示、`CategoryNav` 侧边分类导航、`Pagination` 分页

---

### 3. UI 重构 — 仿 B 站风格 + 统一设计 Token

**技术栈**：CSS Custom Properties（设计 Token）、Vue 3 scoped CSS、Google Fonts（Noto Serif/Sans SC）

**实现方式**：
- 在 `design-tokens.css` 中定义全局 CSS 变量：颜色体系（暖米色主调 + 金色点缀）、圆角、间距、阴影、字体栈
- `BannerHeader` 组件实现仿 B 站的全屏视差横幅：大背景图 + 渐变叠加层 + 居中搜索栏 + tagline 轮换
- 将 `BannerHeader` 提升至 `App.vue` 保持跨路由常驻，避免切换闪烁
- 三个前端应用（reader / admin / login）共享同一套设计 Token，视觉统一

---

### 4. 书评系统

**技术栈**：Spring Boot JPA、Vue 3、递归组件

**实现方式**：
- 实体：`Review`（自引用 parent_id 支持嵌套回复）+ `ReviewLike`（用户点赞记录）
- `ReviewService` 支持：CRUD、点赞切换（toggle 模式防重复）、多级嵌套回复递归查询
- 前端 `ReviewSection` 集成 `ReviewItem` 递归渲染嵌套回复树
- 支持按时间 / 热度排序

---

### 5. 举报与审核系统

**技术栈**：Spring Boot JPA、Vue 3 Modal 组件、管理员后台

**实现方式**：
- 实体：`Report`（target_type 区分 review/note，status 跟踪处理状态）+ `Notification`（关联举报与回复）
- 前端：`ReportDialog` 弹窗供用户提交举报；管理员 `ReportManagement` 页面列表展示及处理
- 重复举报拦截：Service 层捕获 `DataIntegrityViolationException`，返回友好提示
- 审核员收到通知后可标记"已处理"或"已驳回"

---

### 6. 通知系统与角标实时更新

**技术栈**：Spring Boot、Vue 3、轮询

**实现方式**：
- 通知实体关联 `user_id`（接收者）、`type`（回复/举报/系统）、`is_read` 状态
- 管理员待处理通知数通过 `GET /api/admin/reports/pending-count` 接口获取
- 前端 30 秒轮询更新未读通知数，渲染红色角标
- Inbox 页面集中展示所有通知，点击跳转至对应内容

---

### 7. 对话视图（会话模式）

**技术栈**：Vue 3 Composition API、CSS Grid、递归组件

**实现方式**：
- 当某条书评的回复数 ≥ 6 时，展示"查看对话"入口
- `ConversationModal` 以线程模式全屏展开该评论的所有嵌套回复
- 每条回复携带 `@username` 目标标签，标明回复对象
- 递归嵌套渲染通过 `ReviewItem` 组件自引用实现

---

### 8. EPUB 上传与阅读器

**技术栈**：Spring Boot MultipartFile、epub.js、Vue 3

**实现方式**：
- 后端：`Book` 实体新增 `epubData` 字段（MEDIUMBLOB），支持 Multipart 上传并解析 EPUB 元数据
- 提供 `GET /api/books/{id}/epub` 公开端点供阅读器加载
- 前端 `BookReader` 使用 epub.js 渲染：章节导航、进度追踪、阅读位置记忆
- 跳过封面页，直接定位首章正文；移除 iframe sandbox 以支持文本选中

---

### 9. 批注与书余（Siyu / 笔记）系统

**技术栈**：Spring Boot JPA、Vue 3、Canvas/SVG、epub.js hooks

**实现方式**：
- 实体：`Note`（关联 book/user，含选中文本、批注内容、是否发布）+ `NoteLike`
- 批注流程：阅读器内选中文本 → `rendition.hooks.content` 注入高亮样式 → 弹出批注面板（支持保存/取消）
- 书余（Siyu）：已发布的笔记按书籍展示，支持排序、点赞、回复、删除
- 余音（Yuyin）：可拖拽的星空图谱可视化——中心为用户节点，已发布笔记作为环绕星点，支持缩放和平移
- "已入余音"徽章：已发布笔记在 Siyu 中显示特殊标识

---

### 10. 数据存储重构 — 封面与 EPUB 文件管理

**技术栈**：Spring Boot、MySQL MEDIUMBLOB、FormData、Blob URL

**实现方式**：
- 移除 `coverUrl` 字段，改为 `coverData`（MEDIUMBLOB）直接存储封面图片
- 移除独立的 `uploadCover` API，封面随 `createBook/updateBook` 的 FormData 一起提交
- `hasCover` 布尔字段替代 URL 判空，前端用 Blob URL 渲染封面
- 新增 `DataMigrationRunner` 将存量文件迁移至 `shuhai/` 备份目录
- 移除 `FileService`、`FileUploadConfig` 等旧上传架构

---

### 11. 自动退登（Token 过期处理）

**技术栈**：axios 响应拦截器、Vue 3 Composition API、localStorage

**实现方式**：
- axios 响应拦截器统一捕获 401 状态码
- 清除 localStorage 中的 token 和 username
- 弹窗提示"登录已过期，请重新登录"
- 延迟 1.5 秒后自动跳转至登录页

---

### 12. 管理员待审核角标实时更新

**技术栈**：Spring Boot、Vue 3、setInterval 轮询

**实现方式**：
- 后端 `GET /api/admin/reports/pending-count` 返回 Integer
- 前端 30 秒定时器轮询该接口，更新 header 角标数字
- 角标仅在 count > 0 时显示

---

### 13. 登录页背景雨滴动效

**技术栈**：Canvas 2D API、requestAnimationFrame、ResizeObserver、Vue 3 Composition API

**实现方式**：
- 新增 `RainCanvas.vue` 组件，使用 Canvas 逐帧绘制雨线
- 每条雨线沿对角线方向斜向飘落（CSS 像素坐标系），通过 `linearGradient` 呈现"顶部淡→底部深"的书法笔触感
- 雨滴属性（长度、速度、透明度、粗细）在合理范围内随机化，产生自然层次
- `ResizeObserver` 自动适配窗口缩放
- `visibilitychange` 事件在标签页隐藏时暂停动画，节省资源
- `devicePixelRatio` 适配 HiDPI 屏幕
- 组件以 `z-index: 1` 插入登录页背景图和遮罩层之间

---

### 14. 登录/注册卡片 3D 翻转效果

**技术栈**：CSS 3D Transforms（perspective、transform-style: preserve-3d、backface-visibility、rotateY）、Vue 3 Grid 布局

**实现方式**：
- 单张卡片，前后两面通过 CSS Grid 的 `grid-template-areas` 叠加
- 正面（登录）和背面（注册）均为 `.flip-face`，背面预先 `rotateY(180deg)`
- 点击"立即注册" / "去登录"触发 `flipped` 状态切换，`.flip-inner` 执行 `rotateY(180deg)` 过渡
- `backface-visibility: hidden` 确保翻转时只显示朝向用户的那一面
- 过渡 0.6s 缓出曲线，Logo 保持在外层不翻转

### 15. 书架功能

**技术栈**：Vue 3、localStorage、Vue Router

**实现方式**：

- 书架页面以网格展示已收藏图书的封面、标题
- 支持从书架移除，点击跳转至详情页

### 16. 首页 Tagline 轮换

- 移动端隐藏 tagline 以节省空间

### 17. 导航与路由体系

**实现方式**：

- admin-app 路由：图书管理 `/`、举报管理 `/reports`
